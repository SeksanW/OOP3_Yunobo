HEAP sort:Algorithm Description and Complexity Analysis

Description: 
A heap sort is an algorithm that sorts data in a structure called a heap.
Think of a tournament bracket where the you have winners and losters, losers goes to the bottom and winners goes to the top.
The index is structured from top down and left to right, so at the very top you will have index 0, fallow by index 1 on the row below starting from the left, then index 2 and so on.
Once the highest number is crowned it then gets taken out of the system and gets locked up in a spot where the heap is rebuilt to crown the next hightest number and so on till there is none left.
Finishing the process and aligning the numbers in order.


Pseudocode:

Phase one Build heap: (example data:[3,2,5,9,8,7] 
	1: You buld a bracket starting with as shown    __________3_________
	   2 is index 0, 3 is 1 and 5 is 2				______2________5_____
													___9______8________7_
													
	2: Then starting at number 2 who will be compared to the bottom 8 and 9
	   since 9 is bigger than 2 so they swap places, at the same time 5 is less than 7 so it gets swapped.
	   leaving 9 and 7 fighting for the top spot, 9 being the larger number takes the spot of number 3 and egts crown.
	3: A at the end pf phase one the largest number should sit at index 0
	
Phase 2 extracting:
	4: Swap index 0 with the last index and locks that slot, so now the data would look [3,7,2,8,5] with [9]
	5: The slot that 9 occupy is now considered a sorted slot.
	6: Calls for heapify on the unsorted slots then uild another heap bracket to sort for the next largest number.
	7: reapeating till all slots are locked.

Heapify:
	8:looking at the current a number will swap positions with the 2 bottom left or rigth slots if their number is lower 


Complexity Analysis

Best-case time complexity:		O(n log n)
Average-case time complexity:	O(n log n)
Worst-case time Complexity:		O(n log n)
Space complexity:				O(1)

Heap is always running 0(n log n) so it will have a really good time efficientcy
